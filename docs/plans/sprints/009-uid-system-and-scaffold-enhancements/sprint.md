# Sprint 009: UID System and Scaffold Enhancements

## Goals

Implement a UID generation system that assigns stable unique identifiers to
courses, modules, lessons, and exercises. Enhance the scaffolding pipeline to
create Tier 3-4 mirror directories (repo-root `lessons/`, `projects/`
alongside `docs/docs/`), integrate `.devcontainer` configuration generation,
and produce explicit `nav` sections in `mkdocs.yml` from the course structure.

## Problem

Curik-scaffolded courses currently have no stable identity mechanism. When
courses are referenced across systems (registries, LMS imports, analytics),
there is no guaranteed unique identifier to link a course, module, or lesson
back to its source. File paths change when courses are reorganized, so path-
based identity is fragile.

Additionally, the scaffolding pipeline has three gaps for Tier 3-4 courses:

1. **No mirror directories.** Tier 3-4 courses need repo-root directories
   (`lessons/`, `projects/`) for student-facing code that lives outside the
   MkDocs `docs/` tree. Currently, `scaffold_structure` only creates
   directories under `docs/docs/`.

2. **No `.devcontainer` integration.** `templates.py` already has
   `get_devcontainer_json()` but nothing in the scaffolding flow calls it.
   Tier 3-4 courses need a `.devcontainer/devcontainer.json` generated
   automatically so students can open the repo in a dev container.

3. **No explicit nav.** `get_mkdocs_yml()` generates a config without a `nav`
   section, relying on MkDocs auto-discovery. This produces unpredictable
   ordering and prevents intentional navigation structure. The nav should be
   generated from the course structure so lesson order matches the spec.

## Solution

1. **UID module (`curik/uid.py`).** Two generators:
   - `generate_course_uid()` returns a UUID4 string for course-level identity.
   - `generate_unit_uid()` returns an 8-character base62 string
     (a-z, A-Z, 0-9) for modules, lessons, and exercises.

2. **UID integration into scaffolding.** `scaffold_structure` and
   `create_lesson_stub` write a `uid` field into YAML frontmatter of every
   generated Markdown stub. `course.yml` gains a `uid` field populated by
   `generate_course_uid()` at init time.

3. **Tier 3-4 mirror directories.** `scaffold_structure` detects the course
   tier from `course.yml`. For Tier 3-4, it creates repo-root `lessons/`
   and `projects/` directories that mirror the module structure under
   `docs/docs/`, giving students a code workspace outside the docs tree.

4. **`.devcontainer` integration.** `scaffold_structure` calls
   `get_devcontainer_json()` for Tier 3-4 courses and writes
   `.devcontainer/devcontainer.json` into the repo root. The language is
   read from `course.yml` (defaulting to `python`).

5. **Explicit nav generation.** A new function `generate_nav` in
   `scaffolding.py` builds a `nav` list from the course structure and
   injects it into the `mkdocs.yml` output. `get_mkdocs_yml()` gains an
   optional `nav` parameter that, when provided, is serialized into the
   config.

## Success Criteria

- `generate_course_uid()` returns a valid UUID4 string.
- `generate_unit_uid()` returns an 8-character string using only base62
  characters, with no collisions in 10,000 sequential calls.
- Scaffolded lesson stubs contain `uid:` in YAML frontmatter.
- `course.yml` generated by `init_course` contains a `uid:` field with a
  UUID4 value.
- For a Tier 3 course, `scaffold_structure` creates both `docs/docs/`
  lesson directories and repo-root `lessons/` mirror directories.
- For a Tier 3 course, `scaffold_structure` writes
  `.devcontainer/devcontainer.json`.
- Generated `mkdocs.yml` contains an explicit `nav:` section matching the
  course module/lesson structure.
- All existing tests continue to pass.
- New unit tests cover UID generation, frontmatter injection, mirror
  directory creation, devcontainer output, and nav generation.

## Scope

### In Scope

- New `curik/uid.py` module with `generate_course_uid()` and
  `generate_unit_uid()`.
- Modification of `scaffold_structure` to write UIDs into frontmatter, create
  mirror directories for Tier 3-4, and write `.devcontainer/devcontainer.json`.
- Modification of `create_lesson_stub` to accept an optional `uid` parameter
  and write it into frontmatter.
- Modification of `get_mkdocs_yml()` to accept an optional `nav` parameter.
- New `generate_nav()` function in `scaffolding.py`.
- Addition of `uid` field to `course.yml` template in `templates.py` and
  `init_course` in `project.py`.
- Unit tests for all new and modified functions.

### Out of Scope

- UID persistence or deduplication across multiple scaffold runs (UIDs are
  generated fresh; idempotency is not required).
- Registry or LMS integration that consumes UIDs (future sprint).
- Changes to validation logic to check for UID presence (future sprint).
- Tier 1-2 mirror directories (Tier 1-2 courses have no student code).
- Custom devcontainer configurations beyond the language-based template.

## Test Strategy

**Unit tests** for the UID module:

- `generate_course_uid` returns a valid UUID4 (parseable by `uuid.UUID`).
- `generate_unit_uid` returns exactly 8 characters, all in `[a-zA-Z0-9]`.
- Uniqueness: 10,000 calls to `generate_unit_uid` produce 10,000 distinct
  values.

**Unit tests** for scaffolding enhancements:

- `scaffold_structure` with a Tier 3 structure: verify `docs/docs/` dirs
  and repo-root `lessons/` dirs both exist.
- `scaffold_structure` with Tier 3: verify `.devcontainer/devcontainer.json`
  is written and is valid JSON.
- `scaffold_structure` with Tier 1: verify no mirror dirs and no
  `.devcontainer/` created.
- `create_lesson_stub` with `uid` parameter: verify frontmatter contains
  `uid:` field with the provided value.
- `create_lesson_stub` without `uid`: verify stub is generated without
  frontmatter (backward compatible).

**Unit tests** for nav generation:

- `generate_nav` from a sample structure returns the expected nested list.
- `get_mkdocs_yml` with `nav` parameter includes a `nav:` section.
- `get_mkdocs_yml` without `nav` parameter omits the `nav:` section
  (backward compatible).

**Integration tests**:

- End-to-end: `init_course` followed by `scaffold_structure` for a Tier 3
  course produces a complete directory tree with UIDs, mirror dirs,
  devcontainer, and explicit nav.

## Architecture Notes

- `curik/uid.py` has no dependencies beyond the Python stdlib (`uuid`,
  `secrets`, `string`).
- UID generation is stateless -- no database or file-based counter. UUID4
  provides sufficient uniqueness for course IDs. 8-char base62 provides
  62^8 (218 trillion) possible values, which is more than sufficient for
  unit-level IDs within a course.
- YAML frontmatter in lesson stubs uses a minimal format: three-dash
  delimiters with `uid: <value>` as the only field. This is compatible
  with MkDocs Material's meta plugin and standard Markdown frontmatter
  parsers.
- Mirror directories use the same module/lesson naming as `docs/docs/` to
  maintain a predictable mapping between documentation and code.
- The `nav` parameter to `get_mkdocs_yml` is a list of dicts matching the
  MkDocs nav schema: `[{"Module Name": ["lessons/01-hello.md"]}]`.
